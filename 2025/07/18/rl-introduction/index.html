强化学习基础介绍

ps. 本来不是研究强化学习这方面的知识，但奈何强化学习已经在各顶会中崭露头角，本科和硕士期间都在学校简单接触过，但是也是学完就扔了，因此打算写成博客来系统性学习记录一下。

本系列主要参考：

  • 西湖大学赵世钰老师《强化学习的数学原理》
  • OpenAI 官方强化学习课程

下面就开始进入强化学习的世界吧 ！！！

1. 什么是强化学习

  • 一句话概括：智能体 (agent) 根据环境 (environment) 反馈的奖励 (reward），调整自己的行为策略 (policy)，以实现长期目标的最大化。
  • 简单举例：训练一只狗学会坐下。每当狗狗坐下时，你会给它奖励（比如小零食）；如果它没有按照指令做，你不给奖励（甚至轻微惩罚）。狗狗通过多次尝试，逐渐学会了“坐下”这个动作能带来奖励，所以以后更愿意这样做。
  • 强化学习 vs 监督学习：
    • 监督学习：Input-Output Pairs 数据格式，学习两者之间的映射关系。
    • 强化学习：数据是智能体和环境交互过程中“生成”的，这些数据通常被称为 “经验 (Experience)” 或 “轨迹 (Trajectory)”。

2. 基础概念

​

强化学习通常由以下几个核心要素组成：

  • 智能体 (Agent)：学习和做决策的主体。
  • 环境 (Environment)：智能体进行交互的对象。当智能体采取动作后，环境会反馈新的状态和奖励。
  • 状态 (State, s)：环境在某个时刻的具体情况。比如，棋盘的当前布局、机器人当前的位置等；也可能是一个图像、一个向量、或一组离散变量。
  • 动作 (Action, a)：智能体在某一状态下可以采取的行为选择。
  • 策略 (Policy, π)：智能体在每个状态下选择动作的规则或概率分布，即状态到动作的映射（输入状态，输出动作）。可以是确定性的或随机的。
  • 奖励 (Reward, r)：环境对智能体采取某个（单个）动作后的 即时 反馈。取决于当前状态、刚刚采取的行动以及下一个状态。
  • 回报 (Return, R/G)：从当前时刻开始，未来能获得的所有奖励的 累计 和。
  • 价值函数 (Value Function)：衡量某个状态（或状态-动作对）长期回报好坏的函数。常见的有状态价值函数 $V(s)$ 和动作价值函数 $Q(s,a)$：
    • 状态价值函数 $V^{\pi}(s)$：在固定策略 $\pi$ 下，从状态 $s$ 出发以后，智能体期望获得的累计奖励。
    • 动作价值函数 $Q^{\pi}(s,a)$：在固定策略 $\pi$ 下，智能体在状态 $s$ 采取特定动作 $a$ 后，再继续按照策略 $\pi$ 行动时所能获得的累计奖励。

这里对奖励 (Reward)，回报 (Return)，价值函数之间的关系做个详细解释：

项目 奖励 Reward ($r$) 回报 Return ($G_t$) 价值函数 Value Function ($V^\pi(s)$, $Q^\pi(s,a)$)
定义 智能体与环境交互时，每一步获得的即时反馈 从当前时刻起，未来所有奖励的累计和（通常带折扣） 从某状态（或状态-动作对）出发，按策略未来回报的期望值
数学表达 $r_{t+1}$ $G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots$ $V^\pi(s) = \mathbb{E}_\pi[G_t \mid s_t = s]$
作用范围 只影响当前这一步 覆盖从当前到未来所有步骤 评价一个状态或状态-动作对的“长期好坏”
依赖对象 当前状态和动作 当前及未来所有奖励 策略、状态（和动作）以及所有可能的未来轨迹
是否有随机性 单步直接反馈，随环境变化 每次经历后都能算出一个具体值，可能不同 多次经历的平均结果（期望），理论上不随采样波动
举例 当前位置走一步，得到 $-1$ 迷宫从起点出发本轮累计得分 $G_0=81$ 起点采用策略 $\pi$ 平均每次能拿 $V^\pi(s_0)=82$
强化学习目标 不是最终优化目标，只是反馈信号 不是直接优化目标，是中间变量 最终优化目标：最大化价值函数（即最大化期望回报）

总结：奖励是单步反馈，回报是一条轨迹的累计奖励，价值函数是回报的期望值，也是强化学习要优化的目标。

3. Markov Decision Processes, MDP

正式介绍 MDP 之前，首先明确一个点：使用 MDP 进行强化学习建模需要依赖于环境，属于经典的 model-based 方法，这一点后面会具体说明。

  • 定义：A sequential decision problem for a fully observable, stochastic environment with a Markovian transition model and additive rewards is called a Markov decision process，即首先 MDP 是一个 序列决策 问题，其次是 完全可观测、具有马尔可夫转移模型，并且 伴有奖励/惩罚机制。

  • 转移模型 (Transition model, T)：描述当智能体在某个状态 $s$ 下采取动作 $a$ 后，会以什么概率转移到下一个状态 $s’$，这个概率可以使用 $P(s’\mid s,a)$ 表示。

  • 数学形式：
    $$
    \langle S, A, R, P, \rho_0 \rangle
    $$

    • S 表示所有有效状态的集合
    • A 表示所有有效动作的集合
    • R: S×A×S→R 是奖励函数，其定义为 $r_t = R(s_t, a_t, s_{t+1})$
    • P: S×A→P(S) 是状态转移概率函数，其中 $P(s’ \mid s, a)$ 表示如果你处于状态 $s$ 并采取动作 $a$，则转移到状态 $s’$的概率
    • ρ0 是初始状态分布。
